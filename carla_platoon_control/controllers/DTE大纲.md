================================================================
论文大纲：基于实时通信延迟的智能网联车自适应编队控制
================================================================
【摘要】
本文研究结构化道路低速环境下智能网联车（CAV）的纵向编队控制问题。
针对V2X通信延迟实时波动导致的间距策略失配、鲁棒集保守性不足
以及周期性控制带来的计算与通信资源浪费问题，
提出一种以实时通信延迟为驱动的自适应编队控制框架。

在间距策略层，提出基于实时通信延迟的动态时距（DTH）方法，
将V2X实测单向延迟实时映射至期望间距，延迟越大间距越保守，
有效补偿信息时效性下降对编队安全性的影响。

在控制器设计层，提出自适应Tube模型预测控制（TubeMPC）方法，
Tube大小随实时通信延迟动态调整，延迟增大时Tube扩张以增强鲁棒性，
延迟减小时Tube收缩以提升控制精度。

在触发机制层，提出事件触发与TubeMPC协同设计方法，
以Tube余量实时调节触发阈值，三个模块统一由通信延迟驱动，
形成闭环协同框架，在保证编队稳定性的同时有效降低计算与通信开销。

基于Lyapunov理论严格证明了闭环系统的递归可行性与编队稳定性，
并通过排除Zeno行为保证事件触发机制的物理可实现性。
仿真与实车实验结果表明，所提方法在低速结构化道路场景下
具有良好的编队跟踪精度、鲁棒性与资源利用效率。

================================================================
一、问题描述
================================================================

1.1 车辆纵向动力学模型
- 二阶模型：状态量为里程s_i(t)和速度v_i(t)
- 控制输入为加速度指令u_i(t)
- 离散化：前向欧拉法，采样周期Ts=0.1s

1.2 编队间距策略
- 基于CTH的间距误差定义（第一篇基础）
- 指出固定CTH在V2X延迟波动下的局限性
- 引出DTH的必要性

1.3 通信模型
- V2X单向通信拓扑：前车→后车
- 通信延迟τ(t)的定义与测量方式（RTT/2或单向时间戳）
- 延迟范围标定：基于实车数据（待补充）

1.4 扰动模型
- 执行器误差作为系统扰动w(t)
- 扰动界：W={w:|w|≤0.5 m/s²}（实车数据95%分位标定）

1.5 控制目标
- 间距误差e_s(t)→0
- 速度误差e_v(t)→0
- 满足安全约束、物理约束
- 减少通信与计算开销

================================================================
二、方法设计
================================================================

--------------------------------------------------------------
创新点1：基于实时通信延迟的动态时距（DTH）
--------------------------------------------------------------

期望间距：
d_ref(t) = h(t) · v(t) + d_0

动态时距：
h(t) = h_0 + γ · τ(t)

约束：
h(t) ∈ [h_min, h_max] = [0.6, 1.5] s

参数：
- h_0 = 0.8s（标称时距）
- γ = 3.3（延迟权重）
- d_0 = 1.5m（静止间距）
- τ(t)：V2X实测单向延迟

间距误差重新定义：
e_s,i(t) = (s_{i-1}(t) - s_i(t)) - d_ref(t)

代码改动：
- 将固定headway=0.6替换为h(t)=h0+gamma*tau(t)
- 加入get_v2x_delay()函数获取实时延迟

--------------------------------------------------------------
创新点2：基于通信延迟的自适应Tube设计
--------------------------------------------------------------

标称系统：
x̄_{i}(k+1) = A_d · x̄_i(k) + B_d · ū_i(k)

实际系统：
x_i(k+1) = A_d · x_i(k) + B_d · u_i(k) + w(k)

误差动态：
e_i(k+1) = A_d · e_i(k) + w(k)，e_i = x_i - x̄_i

自适应tube大小：
Ω(t) = Ω_0 + β · τ(t)

- Ω_0：基础tube，由扰动界W离线计算
- β·τ(t)：延迟附加项

TubeMPC约束收紧：
状态约束从[x_min, x_max]收紧为[x_min+Ω(t), x_max-Ω(t)]

代码改动：
- 重写solveMpc()，加入tube收紧约束
- omega随tau(t)实时更新

--------------------------------------------------------------
创新点3：事件触发与TubeMPC协同设计
--------------------------------------------------------------

Tube余量定义：
ρ(t) = 1 - ||e_i(t)|| / ||Ω(t)||

触发函数：
f(t) = ||e_i(t)|| - σ · ||Ω(t)||

触发条件：
f(t) > 0 → 触发，重新求解MPC并通信
f(t) ≤ 0 → 不触发，保持上次控制量a0_last

三者耦合逻辑：
τ(t)↑ → h(t)↑（间距保守）
       + Ω(t)↑（tube扩张）
       + σ·||Ω(t)||↑（触发阈值增大，更敏感）

代码改动：
- 加入should_trigger()判断函数
- pub_callback_pid()中加触发分支
- 保存a0_last供非触发时刻使用

================================================================
三、仿真验证（CARLA）
================================================================

平台：Ubuntu 20.04 + Python 3.8 + CARLA 0.9.13 + ROS2 Foxy

场景设置：
- 三辆车编队，Town04地图
- 路线B（直道+弯道）或路线C（环形多弯道）
- 领航车执行变速profile：0→3m/s→2m/s（过弯）→3m/s
- 通信延迟：Python注入随机延迟，范围10ms~100ms

对比基线：
- Baseline1：固定CTH + 周期性MPC（传统方法）
- Baseline2：固定CTH + 事件触发MPC
- Proposed：DTH + 自适应TubeMPC + 事件触发协同

评价指标：
- 间距误差RMSE
- 速度误差RMSE
- 触发率（触发次数/总周期数）
- 最小间距（安全性）
- 控制输入平滑度
